# 04. 顔・表情システムと髪システム

## 顔 = デカール哲学(Roblox的自由)

顔パーツはボーンを持たない。**head座標系に自由配置される2Dデカール**であり、表情は「シェイプ差し替え+トランスフォームオフセット」で駆動する。Robloxの「顔は自由なテクスチャ」という精神を、パーツ分離によって表情駆動可能にした形。

点目でも、リアルな目でも、記号的な「3」の口でも、システムは関知しない — **どんな絵でもシェイプセットの枠に入れれば表情が動く**。

## 顔スロットとシェイプセット

各スロットは「シェイプセット」= 名前付きバリアントの集合を持つ。

| スロット | z | 必須シェイプ | 推奨追加シェイプ |
|---|---|---|---|
| browL/R | 78 | neutral | up / down / angryIn / sadOut / worried |
| eyeL/R | 74 | open | closed / half / happy(`^`) / wide |
| pupilL/R | 75 | default | (シェイプではなくoffsetで視線制御) |
| nose | 72 | — (任意スロット) | |
| mouth | 73 | neutral / smile / openSmile / frown / flat | open / sad-open / 母音6種(将来) |
| cheek | 71 | — (任意) | blush |
| faceFx | 79 | — (任意) | 汗 / 涙 / 怒りマーク / 影(漫画記号スタンプ。Phase 6) |

- シェイプ間の切替は**スワップ**(モーフではない)。フラットデザインではスワップの方がキレが出る
- 各シェイプはパートエディタと同じプリミティブ作画で作成
- L/Rはミラー複製で片側だけ作ればよい

表情 = スロット毎のシェイプ選択+オフセットの組み合わせ。v1標準10種:

| プリセット | brow | eye | mouth | 補足 |
|---|---|---|---|---|

| smile | neutral | open | smile | |
| laugh | up | happy | openSmile | |
| sad | sadOut | half | frown | |
| cry | sadOut | closed | sad-open | faceFx:涙(将来) |
| angry | angryIn | open | frown | |
| surprised | up | wide | open | 眉offsetY上 |
| worried | worried | open | flat | |

- プリセットは**キャラ毎に上書き可能**(このキャラの「怒り」はこの組み合わせ、と再定義できる)
- オフセット: 眉のY位置・回転、口のスケール等を微調整値として持てる

## 視線(pupil offset)

瞳はシェイプ差し替えではなく **eye境界内のoffset(x, y)** で制御。「カメラ目線」「左の相手を見る」「目を泳がせる」がワンパラメータで表現でき、紙芝居の会話シーンの説得力が大きく上がる。クリップ/タイムラインからキー指定可能。

## まばたき

- `open → closed → open`(計100〜140ms)を **2〜6秒間隔(シーンseedによる決定論的乱数)** で自動再生
- 表情が closed/happy 系のときは抑制
- キャラ毎にON/OFF・頻度係数

## 口とリップフラップ

- **v1(Phase 5)**: 音声RMS → `closed / half-open / open` の3段階を**12fps量子化**でパタパタ切替(紙芝居らしい軽快さを意図した仕様)
- **将来(Phase 6)**: 日本語母音ビゼーム `a / i / u / e / o / n` の6シェイプ対応。音素解析(簡易: 音量+ピッチ、本格: forced alignment)でシェイプ列を生成。日本語ナレーション前提なので5母音+んで十分

## 髪システム(Live2D的3層構造)

### 層構造

| 層 | z | 役割 |
|---|---|---|
| hairBack(後ろ髪) | 体より奥(z=10) | 後頭部〜背中。走りで最も大きくなびく |
| hairMid(中間髪) | 顔より手前(z=80) | 横髪・もみあげ・輪郭まわりのボリューム |
| hairFront(前髪) | 顔の最前(z=85) | 額の上。目に被ってもよい |

### ストランド

各層は1個以上の**ストランド**(揺れの単位)を持つ。
`ストランド = シェイプ群 + 根元ピン(origin) + 物理パラメータ`

例: ロングヘア = hairBack 2ストランド(左右の房)+ hairMid 2(横髪)+ hairFront 1。ポニーテール = hairBack 1(結び)+ …

### 物理モデル(Live2D風振り子)

ストランド毎に1〜2節の**減衰振り子チェーン**。根元ピンを支点に回転、2節目で先端のしなりを表現。

```
torque = -stiffness × (θ - θrest) - damping × ω + inertia × (支点の加速度の慣性項) + gravity寄与
```

| パラメータ | 範囲 | 意味 |
|---|---|---|
| stiffness | 0..1 | 復元の強さ(高い=硬い短髪) |
| damping | 0..1 | 減衰(低い=いつまでも揺れる) |
| inertia | 0..1 | 頭の動きへの追従度(揺れ幅に相当) |
| maxAngle | deg | 振れ角クランプ(破綻防止) |
| gravity | 0..1 | 垂れ下がり寄与(ポニテ向け) |
| segments | 1..2 | 節数(2で先端のしなり) |

**プリセット**: ショート / ボブ / ロング / ポニーテール / アホ毛(極端値で遊ぶ)。数値を直接いじらなくても選ぶだけで成立させる。

### 駆動と決定論

- 入力はheadボーンの**ワールド速度・加速度**(=root移動+体の回転の合成)。クリップを再生すれば勝手に揺れる
- dt=1/120固定サブステップ、シーン先頭からシミュレート(02の決定論原則)
- 期待挙動の品質基準:
  - 歩き: 軽い周期揺れ
  - **走り: 後方へなびく+上下バウンス(本ツールの看板品質)**
  - 急停止: オーバーシュートして戻る
  - ジャンプ・落下: ふわっと浮き上がる
- シーン単位のグローバル風パラメータはPhase 6

### 揺れ対象の拡張

同じストランド機構を**髪以外にも適用可能にする**: スカーフ、ネクタイ、ロングスカート、ウサ耳。スロットとしては `accessory(physics付き)` で汎用化。実装は髪と完全共通。
