# 08. ロードマップ(フェーズ計画)

各フェーズは「動くものが出て、受入条件で検証できる」単位。モデル列は 09 のモデル戦略と連動。

| Phase | 内容 | 受入条件 | 主担当 | fable5の出番 |
|---|---|---|---|---|
| **0. 基盤** | scaffold(Vite/React/TS/Pixi)、core-modelスキーマ、コマンド+undo、保存/読込(FS Access) | 空プロジェクトを保存→再読込→undo/redoが動く | sonnet | アーキ設計レビューのみ |
| **1. キャラ作成コア** | humanoid-v1、ピン機構、パートエディタ(プリミティブ作画・ミラー複製・SVGインポート)、パレットスロット、テンプレ1体 | テンプレ改変で新キャラを作成→保存→再読込→静止ポーズ表示 | **fable5** | リグ/ピン/座標変換の設計&実装、ゴースト下絵UX、視覚検証 |
| **2. 表情と髪** | 顔シェイプセット、表情プリセット合成、視線、まばたき、3層髪+振り子物理 | 表情10種切替+「走り(仮クリップ)で髪が躍動するGIF」が出せる。コンタクトシートCLI完成 | **fable5** | 物理の挙動チューニング、表情合成、コンタクトシート視覚検証 |
| **3. アニメランタイム** | クリップ評価器、プリセット10本、遷移、flip、ハンドシェイプ、合成パイプライン | 1キャラでwalk→run→point→talkを途切れず再生、反転も正常 | fable5(評価器・クリップ数学)+ sonnet(プリセットデータ量産) | walk/runの見た目品質判定(コンタクトシート+短尺動画フレーム) |
| **4. シーンエディタ** | ステージ操作、右クリックメニュー、スナップ、シーンサムネ帯、タイムライン、enter/exit、テキスト/吹き出し、アセットパネル | 3シーンの紙芝居(キャラ2体+背景+テキスト)を組んで通し再生できる | **opus**(タイムライン/undo統合)+ sonnet(UI量産) | UIスクリーンショットの仕上がり確認(節目のみ) |
| **5. 音声と書き出し** | VO/BGM、トーク+リップフラップ、カメラ、WebCodecs MP4書き出し | 音声付き30秒の動画がMP4で書き出せ、プレビューと一致する | **opus**(エンコードパイプライン)| 書き出しフレームとプレビューの一致検証(pixel-diff越えの差分判定) |
| **6. 仕上げ** | Replace機能、スターターアセットパック(背景10/小道具30)、ダッキング、faceFx(汗・涙)、タイプライター、風 | Replaceでキャラ差し替えても動画が成立。アセット検収完了 | sonnet(アセットSVG量産・機能群)| **アセットパックのコンタクトシート検収**(アートディレクション) |
| **7. 発展** | 母音リップシンク(あいうえお)、ポーズエディタ、プロポーション調整、アプリ内AI機能、Electron化判断 | — | 都度判断 | 母音同期の品質判定、アプリ内AIの設計 |

## マイルストーン

- **M1(Phase 2完了)**: 「自作キャラの髪が走りで躍動する」— 本ツールの存在意義の証明。ここまで最短で行く
- **M2(Phase 5完了)**: 「最初の1本のMP4」— 実用開始ライン
- **M3(Phase 6完了)**: Vyondからの実制作移行

## スターターアセット制作計画(Phase 6)

アセット制作自体もモデルで量産する:

1. fable5が 01 のアートスタイルガイドから**SVG記述規約**(viewBox・パレット・形状語彙)を起こす
2. sonnetが背景・小道具SVGを一括生成(1セッションで10〜15個)
3. コンタクトシート化 → fable5が一括検収(色・シルエット・スタイル統一)→ NGのみ再生成

## 依存関係

```
Phase 0 → 1 → 2 → 3 → 4 → 5 → 6
              └─(2と3は一部並行可: 物理とクリップ評価は独立)
```
