# 11. シーン編集UXオーバーホール(Phase 4c)

06のVyond的編集体験仕様を、Phase 4b完了時点の実装とのギャップを踏まえて具体化する。
機能は揃った(配置・アクション・カメラ・トランジション・右クリック・スナップ)が、**見た目で選べない・ドラッグで調整できない・絵で確認できない**。本仕様はその3点を潰す。

## 目的

1. **視覚的な素材確認**: キャラ・背景・アクション・表情を「名前」ではなく「絵」で選ぶ
2. **直接操作**: 時刻・大きさ・カメラを数値入力ではなくドラッグで調整する
3. **AI台本駆動への布石**: 将来、AIが台本(キャラの行動・表情・カメラフォーカス・場面転換を全記述)を生成し、人間が手直しして仕上げるワークフローの「手直し側の受け皿」を作る

北極星となる受入条件: **3シーンの紙芝居を、座標・時刻の数値入力ゼロ(マウス操作だけ)で組める。**

## 現状の問題(Phase 4b時点)

| 箇所 | 問題 |
|---|---|
| アセットパネル | キャラ・背景・吹き出しがテキストボタンの羅列。絵が見えない |
| シーン帯 | 「シーン1 4.0秒/2要素」の文字のみ。シーンの内容が判別不能 |
| タイムライン | アクションが点チップ(区間が見えない)、全てクリック選択→数値編集。ドラッグ調整不可。enter/exitは3px線 |
| プロパティ | 表情・クリップがネイティブselect。enter/exitもselect。絵の確認はステージで再生するしかない |
| カメラ | x/y/zoomの数値入力のみ。Vyondの「カメラ枠をドラッグ」体験がない |
| ステージ | 拡縮はスライダーのみ(ハンドルなし)、ホバーフィードバックなし |
| 全体 | 素のHTML+インラインstyleでテーマ不統一(黒地に白ボタン)。アイコンなし |

## 設計原則

- **絵 > 名前**: 選択肢は常にサムネイル・アイコン・プレビュー付き。評価器(evaluateScene / buildRenderList)が唯一の絵の源泉 — サムネも本番描画と同じコードで作る(WYSIWYG)
- **ドラッグ > 数値**: 数値入力は残すが補助に格下げ。主操作はステージとタイムライン上の直接操作
- **既存アーキテクチャ不変**: スキーマ・評価器・コマンド層(undo)は完成している。本フェーズは **src/editor/ と src/render/ のみ**を触る。新コマンドが必要な場合も既存のdispatch/mergeKey規約に従う追加のみ
- **台本=時系列の言葉**: 人間にもAIにも、シーンは「t=1.2でハルが驚く」という時系列イベント列として読める/書けるべき(→台本ビュー)

## 機能仕様

### A. サムネイル生成基盤(ThumbnailService)— 本フェーズの土台

`src/editor/thumbs/thumbnail-service.ts` 新設。**単一の共有オフスクリーンPixi Application**(遅延初期化)で逐次レンダリングし、dataURLをキャッシュする。ContactSheetPageに前例あり。

```ts
export class ThumbnailService {
  // キャラ立ち絵(表情・クリップ位相も指定可 — クイックアクションのポーズサムネに使う)
  renderCharacter(char: CharacterDoc, opts?: {
    expression?: string;   // EXPRESSION_PRESETSキー(デフォルトneutral)
    clip?: string; phase?: number;  // クリップ姿勢(デフォルトはレストポーズ)
    face?: boolean;        // true=顔のみクロップ(表情ピッカー用)
    w?: number; h?: number;
  }): Promise<string>;     // dataURL
  // シーンサムネ(t = duration/2 で evaluateScene、背景込み)
  renderScene(project: ProjectDoc, scene: SceneDoc, w?: number, h?: number): Promise<string>;
  invalidateScene(sceneId: string): void;   // doc変更時(300ms debounceで再生成)
  subscribe(cb: () => void): () => void;     // 生成完了通知(Reactはrev++)
}
```

- 生成は1フレーム1枚のキュー処理(rAF逐次)でメインスレッドのジャンクを防ぐ
- キャッシュキー: キャラ=`ref+opts`(キャラdocはセッション中不変扱い、resolver.invalidateで全クリア)、シーン=`scene.id`(invalidateSceneで破棄)
- RenderTexture / 一時Containerは**毎回必ずdestroy**(リーク厳禁)
- サムネは派生データ。**.byp.jsonには一切保存しない**
- 既定サイズ: キャラ72×108 / 顔56×56 / ポーズ72×72 / シーン128×72

### B. アセットパネルの視覚化

- **キャラ**: サムネイルグリッド(2列)。内蔵+保存済(characters/)をThumbnailServiceで立ち絵化。名前は下に小さく。クリックで配置(既存addCharacter)
- **背景**: 画像サムネグリッド — リポジトリ内蔵(assets/generated/)+プロジェクトフォルダ(assets/bg/)。`resolver.getImageUrl`で取得した実画像を`<img>`縮小表示(Pixi不要)。色背景はスウォッチ8色+カラーピッカー。クリックで現在シーンに適用。手入力欄は廃止
- **吹き出し**: 3形状のミニSVGアイコン(インラインSVGで角丸/雲/トゲの輪郭を描く。drawBalloonの再現は不要、記号で十分)
- **テキスト**: 「あ」プレビュー付きボタン1個(将来スタイルプリセットの席)
- セクションは折りたたみ(共通Sectionコンポーネント)

### C. シーン帯の絵コンテ化

- 各シーン = **サムネカード**(128×72のrenderScene + シーン番号 + 長さ秒)。選択中は強調枠、通し再生中はプログレスバー
- **ドラッグ&ドロップで並べ替え**(HTML5 DnDでよい。ドロップ位置にインジケータ。既存moveSceneを再利用 — 複数位置移動になる場合はmoveSceneを`toIndex`版に拡張可)
- カード右クリック: 複製/削除(既存コマンド)
- **シーン間にトランジションボタン**: カードの間に小さな「⇄」チップ。クリックでポップオーバー(カット/フェード/ワイプ/スライド + 長さ)。設定済みは種別アイコン表示(=次シーンのtransitionを編集)
- doc変更でinvalidateScene → debounce再生成(編集中のカクつき防止)

### D. タイムラインの本格化(ブロックモデル)

- **アクションブロック**: 区間 = `[a.t, 次アクションのt)`(最後はシーン末まで)。クリップ名label入り。**ドラッグでt移動**(0.05s刻み+他キー時刻・整数秒へスナップ、mergeKey `act:${elId}:${i}:t`)。moveTo付きは到着時刻まで「▸▸▸」模様+到着点に印
- **表情キー**: ◆マーカー+顔ミニアイコン(ThumbnailServiceのface、またはプリセットlabel頭文字)。ドラッグでt移動
- **enter/exit**: レーン両端のグラデブロック(enter=左端からdelay+dur幅、exit=atからdur幅)。ブロックドラッグでdelay/at、端ドラッグでdur。cutは細線のまま
- **カメラレーン**(キーが1つ以上ある時表示): ◆ドラッグでt移動、クリックで「カメラキー選択」状態→プロパティタブにキー詳細
- **再生ヘッド**: ルーラーから全レーンを貫く縦線。レーン上でも横ドラッグでスクラブ可
- レーンヘッダ: 種別アイコン+名前+🔒。クリック選択(既存)
- ブロック/マーカーのドラッグは**開始値+累積デルタ**方式(複利禁止 — progress.md落とし穴3)
- タイムライン上の操作はすべてundo 1回で戻る

### E. ステージ直接操作の拡充

- **ホバーハイライト**: pointermove(非ドラッグ時)でヒット要素に薄枠(選択枠より細く淡い)
- **拡縮ハンドル**: 選択枠の四隅に□ハンドル。ドラッグでscale変更(対角固定ではなく中心固定で可。Shiftで…等比は元々等比なので不要)。mergeKey既存 `el:${id}:tf`
- **ダブルクリック → クイックアクションポップオーバー**(Vyondのアクション選択の再現):
  - キャラをダブルクリック→ポップオーバーに2タブ: 「アクション」=CLIPS10種のポーズサムネグリッド(renderCharacter {clip, phase:0.4})、「表情」=EXPRESSION_PRESETS10種の顔サムネグリッド
  - クリック→**現在の再生ヘッドtにキー追加**(addAction / addExpressionKey)。タイムラインと連動した「いま・ここに置く」体験
- **カメラモード**(ツールバー📷トグル、Vyondのカメラ操作の再現):
  - ステージに現在t評価のカメラ枠(16:9固定矩形、外側を半透明グレーアウト)
  - 枠内ドラッグ=x/y、四隅ハンドル=zoom(16:9維持、0.5〜4にクランプ)
  - pointerup時: 現在tにキーがあれば更新、なければ追加(updateCameraKey / addCameraKey)
  - プリセットボタン(枠の下に浮かす): 「全景」(960/540/zoom1)/「選択要素に寄る」(要素boundsの中心へ、頭部が画面上1/3に来るzoom≈1.8)— **将来AI台本の `focus: <キャラ>` 指示と同じ実装を共有する**
  - 通常モードのヒットテスト/ドラッグはカメラモード中無効。Esc・再トグルで解除

### F. プロパティパネルの視覚化

- 折りたたみセクション化(配置 / 登場・退場 / アクション / 表情 / 吹き出し / シーン設定)
- **enter/exit**: selectをやめ、アイコン付きセグメントボタン(✂|◐|←|→|↑|↓|◎)+delay/durは小型数値(数値は残してよい箇所)
- **表情**: select→顔サムネグリッド(クイックアクションと同じ部品)
- **アクション行**: クリップ名はポーズサムネ付きドロップダウン(共通部品)。moveTo・時刻の数値は残す(タイムラインドラッグが主、数値は微調整)
- **シーン設定**(未選択時): トランジションをアイコンセグメント化、カメラキー一覧は現状の数値+「このキーへジャンプ」(tへスクラブ)追加
- キャラ選択時に**Replace**ボタン(右クリックと同じ差し替えメニュー)を追加 — 右クリックを知らなくても到達できる

### G. テーマとアイコン

- `src/index.css` 新設、CSS変数でトークン化:
  `--bg-app:#1c1c20 --bg-panel:#26262b --bg-elev:#2f2f36 --border:#3c3c44 --text:#e8e6e3 --text-dim:#98959e --accent:#5B7DB1 --accent-soft:#3d4f6e --warn:#D9704C --danger:#e05656 --radius:6px`
  ステージ周辺だけ明るく(紙芝居の紙色が映える額縁)。8pxスペーシング
- 共通UI部品 `src/editor/ui/`: Section(折りたたみ)/ SegmentedButtons / IconButton / Popover / Thumb(画像+選択枠)。**インラインstyleの都度書きをやめ、この部品とCSSクラスに寄せる**
- **SVGアイコンセット** `src/editor/ui/icons.tsx`: 16×16・stroke=currentColor・約22種(再生/通し/停止/undo/redo/フォルダ/保存/グリッド/カメラ/キャラ/テキスト/吹き出し/背景/ロック/解除/複製/削除/反転/前面/背面/⇄/◆)。絵文字を全廃する

### H. 台本ビュー(AI台本駆動への布石)

右パネルを「プロパティ | 台本」のタブ切替に。台本タブ = **現在シーンの時系列イベントリスト**:

```
0.0  🧍ハル      登場(カット)・歩き → x1500
0.0  💬「おはよう!」 (ハルの上)
1.2  😲ハル      驚き
1.2  📷         ズーム 2.0 へ(〜2.4s)
3.5  🧍ハル      会話A
     ⇄ 次シーンへ: フェード 0.5s
```

- 全要素のenter/action(+moveTo)/expression/カメラキー/トランジションをt昇順に列挙。種別アイコン+キャラ名+内容
- **行クリック → 該当要素を選択+再生ヘッドをそのtへ**(編集対象への最短ジャンプ)
- 吹き出し・テキストの本文は**インライン編集可**(setBalloonProps/setTextProps)。それ以外の行は読み取り+ジャンプのみ(v1)
- 将来(Phase 7): AIが生成した台本JSONを取り込むとこのビューに反映され、人間がここから各行へ飛んで手直しする。**台本JSONスキーマの定義はPhase 7スコープ**(現ProjectDocが実体層。AI向けの「セリフ・ト書き中心の台本層→ProjectDocコンパイル」はその時に設計)。本フェーズはUIの席と「時系列イベント列」という語彙を確立するところまで

> 設計(本仕様+各ハンドオフ)= fable5、実装 = opus/sonnet、検収 = fable5のスクショ確認(節目のみ)

| サブフェーズ | 内容 | モデル | 根拠 |

- **実行順: 4c-1 → 4c-2 → 4c-3 → 4c-4**(直列。全員がScenePage/PropertyPanelを触るため並列はコンフリクトの方が高くつく)
- 着手時にfable5が薄いハンドオフ(読むファイル・受入条件・落とし穴)を1本ずつ発行。本仕様が詳細を持つのでハンドオフはPhase 4bの半分以下の分量でよい
- トークン目安: sonnet 2本 ≈ 150-200k×1 + 80k×1、opus 2本 ≈ 250k×2、fable5(設計済+ハンドオフ4+検収2-3回)≈ 100k。**計 ≈ 0.9M**(Phase 4b実績と同等)

### 検収方式(fable5のトークン節約)

1. 各サブフェーズ完了時、実装エージェントが**ヘッドレスで操作スクリーンショットを撮って exports/ に保存**(progress.mdの「検証の落とし穴」§6-7を必読: setPointerCaptureパッチ・store.doc読み)
2. fable5の目視は2回だけ: **4c-1後**(テーマ+サムネ品質 — 1枚のグリッド合成で)と**4c-3後**(総合操作。シーン帯+カメラ枠+クイックアクションの3状態合成1枚)
3. 4c-2・4c-4はユニットテスト+エージェント自身のヘッドレス確認で通す(タイムラインのドラッグ数学は純関数に切り出してテスト — snap.tsの前例)

### フェーズ受入条件

- **4c-1**: キャラ・背景・表情・登場退場が全て絵/アイコンで選べる。サムネ生成でメモリリークなし(連続100枚生成でheap安定)。テスト全green維持
- **4c-2**: アクション・表情・enter/exit・カメラキーの時刻をすべてドラッグで調整でき、全てundo 1回で戻る。スナップが効く
- **4c-3**: カメラを枠ドラッグだけで設定できる。ダブルクリック→ポーズサムネ→現在tに追加が動く。シーン帯がサムネで判別でき、D&Dで並べ替えられる
- **4c-4**: 台本タブでシーンの流れを通し読みでき、行クリックでジャンプ、セリフをその場で直せる
- **全体(北極星)**: 3シーン紙芝居を数値入力ゼロで構築 → 通し再生 → 保存/再読込で完全復元

## ロードマップ上の位置

Phase 4cとして4bと5(M2: 音声と書き出し)の間に挟むことを推奨。M2を急ぐなら4c-1/4c-2(操作性のコア)だけ先行し、4c-3/4c-4はPhase 5後でもよい — 4c-3のカメラモードと4c-4はPhase 7(AI台本)の直前までに揃っていれば台本駆動に間に合う。
